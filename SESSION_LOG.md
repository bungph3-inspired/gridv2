### Key lessons + memory

- **Paste-output token leak pattern** — `git push -u "<url-with-token>"` plus the user's habitual paste-script-output workflow leaks PATs into chat. Memory saved as `feedback_scan_script_output_tokens.md`. Future safe-push pattern: `git push "<url-with-token>" <branch> 2>&1 | grep -v 'x-access-token'` (no `-u`, output filtered).
- **Two-service systemd architecture** — gridv2.service (Hono API) + gridv2-oddspapi.service (poll worker). Backend code changes affecting both modules (tournament-map, participants) need both restarted.
- **NODE_ENV=production blocks devDeps** — `/etc/gridv2/env` sets it; `source /etc/gridv2/env` then `npm ci` silently skips tsc/drizzle-kit. Always `unset NODE_ENV` for builds.
- **OddsPapi `/v4/participants` returns object** `{id: name}`, not array of objects. jq `.[].participantId` panics with "Cannot index string with string". Use `to_entries` or Python.
- **OddsPapi rate limit** ~1 req/sec on the free tier per endpoint. Probes iterating sports need `sleep 1.5` between calls or a 429 retry with `wait N` parse.
