## Tasks

### Task 1 ‚Äî Remove dead CSS classes left by PR23

- **Context:** PR23 (commit `00c2dbe`) removed the `BS_GAME_META` const +
  `getGameMeta` function + the optional meta-row rendering in
  `buildGameBlock`. The CSS classes that styled those elements are still
  in `src/style.css` and never get applied anywhere now. Strip them so the
  stylesheet stays honest.
- **Acceptance:**
  - All of the following class selectors removed from `src/style.css`,
    including chained variants like `.gbadge.alt`:
    `.gbadge`, `.gseed`, `.gnet`, `.gseries`, `.gmaxw`, `.ginjury`,
    `.ginfo-meta`
  - `grep -RE '\b(gbadge|gseed|gnet|gseries|gmaxw|ginjury|ginfo-meta)\b' src/`
    returns no matches in any file
  - `npx vite build` (from repo root) completes with no new warnings or
    errors compared to `main` HEAD before this PR
  - Diff is removal-only; no rules added, no other classes touched
- **Touch:** `src/style.css`

### Task 2 ‚Äî Personalize the home header

- **Context:** After PR19 (commit `873948c`), the home view shows a static
  `<div class="hv-hdr">Pick a sport</div>`. We have the logged-in player's
  username available ‚Äî show it. When `localStorage.getItem('bs_player')`
  returns a non-empty string, greet them by name. Fall back to the existing
  copy when there is no signed-in player.
- **Acceptance:**
  - `showHome()` in `src/main.js` updates `.hv-hdr` text content before the
    tiles render (no flash of the old text)
  - When `localStorage.getItem('bs_player')` is a non-empty string after
    `.trim()`, the header reads `Welcome back, <username>` where `<username>`
    is passed through `escapeHtml()` (already imported in `main.js`)
  - When the player ID is null, empty, or whitespace-only, the header stays
    as `Pick a sport`
  - Switching to a board view and clicking the BETSIM logo to return to
    home re-runs the personalization ‚Äî the greeting reflects the current
    `bs_player` value, even if it changed
  - No localStorage writes ‚Äî read-only on `bs_player`
- **Touch:** `src/main.js`
